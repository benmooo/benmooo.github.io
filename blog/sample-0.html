Next.js 15 与 MDX 博客搭建指南

使用 Next.js 15 App Router 和 MDX 构建现代化博客系统的完整指南

2024年1月15日

在现代 Web 开发中，Next.js 已经成为构建 React 应用的首选框架。结合 MDX，我们可以轻松创建一个功能强大且易于维护的博客系统。

https://www.google.com

为什么选择 Next.js 15 + MDX？

性能优势

Next.js 15 引入了许多性能优化：

  • Turbo Pack 提供更快的构建速度
  • React Server Components 默认支持
  • 改进的 静态生成 机制

开发体验

MDX 让我们能够在 Markdown 中无缝使用 React 组件：

import { Alert } from '@/components/ui/alert'

<Alert type="info">
  这是一个在 MDX 中使用的 React 组件！
</Alert>

技术栈

我们的博客使用以下技术栈：

  1. Next.js 15 - 核心框架
  2. TypeScript - 类型安全
  3. Tailwind CSS - 样式系统
  4. MDX - 内容管理
  5. Gray Matter - Front Matter 解析

实现步骤

第一步：项目初始化

npx create-next-app@latest my-blog --typescript --tailwind --app
cd my-blog
npm install @next/mdx @mdx-js/react gray-matter

第二步：配置 MDX

在项目根目录创建 next.config.js：

import createMDX from '@next/mdx'

const nextConfig = {
  pageExtensions: ['js', 'jsx', 'mdx', 'ts', 'tsx'],
}

const withMDX = createMDX({
  // 添加你的 MDX 配置
})

export default withMDX(nextConfig)

第三步：创建动态路由

使用 App Router 的动态路由功能：

src/
  app/
    blog/
      [slug]/
        page.tsx
    posts/
      sample-post.mdx

代码高亮

支持多种编程语言的语法高亮：

def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

# 计算斐波那契数列的前10项
for i in range(10):
    print(f"F({i}) = {fibonacci(i)}")

总结

通过 Next.js 15 和 MDX 的结合，我们可以：

  • ✅ 享受现代化的开发体验
  • ✅ 获得出色的性能表现
  • ✅ 保持内容管理的简洁性
  • ✅ 实现组件的无缝集成

这套技术栈非常适合个人博客、技术文档和内容网站的构建。
